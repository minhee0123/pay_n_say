**프로젝트명:** Pay N Say
**버전:** 1.0
**작성일:** 2026-03-18
**패키지 ID:** com.mini.pay_n_say

## 1. 개요

### 1.1 목적
Pay N Say는 가계부 관리와 영어 학습을 결합한 모바일 앱이다. 일상 소비 내역을 기록·관리하면서, 해당 내역을 영어 표현으로 변환하여 자연스럽게 금융/소비 관련 영어를 학습할 수 있도록 한다.

### 1.2 대상 사용자
- 개인 재무 관리가 필요한 사용자
- 소비/금융 관련 영어 표현을 학습하고 싶은 사용자

### 1.3 기술 스택
| 구분 | 기술 |
|------|------|
| 프레임워크 | Flutter (Dart SDK >=3.0.6 <4.0.0) |
| 상태관리 | Riverpod (flutter_riverpod ^2.4.10) |
| 라우팅 | GoRouter (go_router ^13.1.0) |
| 로컬 저장소 | Hive (hive ^2.2.3, hive_flutter ^1.1.0) |
| 테마 | Material 3 |
| 플랫폼 | iOS, Android, macOS |

## 2. 기능 요구사항

### 2.1 가계부 (Ledger) 기능

#### FR-L01: 월간 캘린더 뷰
- **설명:** 월별 캘린더로 일별 지출/수입을 시각적으로 표시한다.
- **상세:**
  - TableCalendar 위젯 기반 월간 달력
  - 지출이 있는 날짜에 dot marker 표시
  - 좌우 스와이프로 월 이동
  - 날짜 선택 시 일별 상세 페이지로 이동
- **우선순위:** 필수

#### FR-L02: 월간 요약 헤더
- **설명:** 현재 월의 수입/지출/잔액을 상단에 표시한다.
- **상세:**
  - 그래디언트 배경 헤더 (accent → accent dark)
  - 수입 합계, 지출 합계, 잔액(수입-지출) 표시
  - 원화(₩) 포맷 (천 단위 콤마)
  - 영어 학습 진입 버튼 (menu_book 아이콘)
- **우선순위:** 필수

#### FR-L03: 일별 상세 페이지
- **설명:** 특정 날짜의 거래 내역을 목록으로 표시한다.
- **상세:**
  - 앱바에 "M월 D일 (요일)" 형식 표시
  - 내역 건수 배지, 수입/지출/잔액 요약 카드
  - 거래 카드 목록 (아이콘, 상호명, 금액, 수입/지출 색상 구분)
  - 내역 없는 날: 빈 상태 안내 메시지
  - FAB로 직접 입력 페이지 이동
  - 카드 탭: 수정 페이지로 이동
  - 카드 롱프레스: 삭제 확인 다이얼로그
- **우선순위:** 필수

#### FR-L04: 직접 입력 (수동 등록)
- **설명:** 사용자가 지출/수입 내역을 수동으로 입력한다.
- **상세:**
  - 지출/수입 토글 선택
  - 입력 필드: 상호명, 카테고리 (드롭다운), 날짜 (DatePicker), 금액 (숫자 키패드)
  - 유효성 검사: 상호명 필수, 금액 필수
  - 금액 입력 시 원화(₩) 접두 표시
  - 저장 버튼 텍스트: "지출 저장하기" / "수입 저장하기"
- **우선순위:** 필수

#### FR-L05: 내역 수정
- **설명:** 기존 거래 내역을 수정한다.
- **상세:**
  - 기존 값을 폼에 미리 채움
  - 앱바 제목: "내역 수정"
  - 저장 버튼: "수정 완료"
  - 수정 후 이전 화면으로 복귀
- **우선순위:** 필수

#### FR-L06: 내역 삭제
- **설명:** 거래 내역을 삭제한다.
- **상세:**
  - 롱프레스로 삭제 다이얼로그 표시
  - "내역 삭제" 제목, 확인 메시지
  - 취소/삭제 버튼
  - 삭제 시 즉시 목록 갱신
- **우선순위:** 필수

#### FR-L07: 데이터 영속화 (Hive)
- **설명:** 거래 내역을 Hive 로컬 DB에 저장한다.
- **상세:**
  - 'transactions' 박스에 Map 형태로 직렬화 저장
  - 앱 재시작 시 자동 복원
  - CRUD 연산 모두 Hive 동기화
- **우선순위:** 필수

### 2.2 영어 학습 (English Learning) 기능

#### FR-E01: 영어 학습 대시보드
- **설명:** 학습 현황과 진입점을 제공하는 메인 페이지.
- **상세:**
  - 학습 진도 카드: "학습한 단어: N/M" + 프로그레스 바
  - 오늘의 영어 표현: 최근 7일 거래 → 영어 문장 변환 (최대 5건)
  - 단어장 / 퀴즈 진입 카드
- **우선순위:** 필수

#### FR-E02: 가계부 연동 영어 표현
- **설명:** 실제 거래 내역을 영어 문장으로 변환하여 표시한다.
- **상세:**
  - 지출: "I spent ₩X on Y." / 수입: "I earned ₩X from Y."
  - 한국어 상호명 → 영어 키워드 매핑 (50+ 키워드)
  - 매핑 실패 시 아이콘 기반 폴백
  - 영어 + 한국어 문장 동시 표시
- **우선순위:** 필수

#### FR-E03: 단어장
- **설명:** 소비/금융 관련 영어 단어를 카테고리별로 학습한다.
- **상세:**
  - 38개 단어 (9개 카테고리: 음식, 카페, 쇼핑, 뷰티, 문화, 교통, 의료, 금융, 수입)
  - 카테고리 필터 칩 (가로 스크롤)
  - 확장 가능한 단어 카드: 영어, 한국어, 예문
  - 북마크 토글
  - 학습 완료(3회 정답) 배지 표시
- **우선순위:** 필수

#### FR-E04: 퀴즈
- **설명:** 4지선다 퀴즈로 영어 단어를 테스트한다.
- **상세:**
  - 10문제 (7문제 단어장 + 3문제 거래 기반)
  - 한→영 / 영→한 혼합 출제
  - 정답/오답 즉시 색상 피드백 (초록/빨강)
  - 결과 화면: 점수, 백분율, 오답 리뷰
  - 정답 3회 누적 시 "학습 완료" 처리
  - 학습 진도 Hive 영속화 ('learning_progress' 박스)
- **우선순위:** 필수

### 2.3 OCR 영수증 스캔 기능

#### FR-O01: 카메라/갤러리 영수증 촬영
- **설명:** 카메라 또는 갤러리에서 영수증 이미지를 선택한다.
- **상세:**
  - image_picker 활용
  - iOS 카메라/사진 라이브러리 권한 설정 완료
  - OCR 엔진은 미구현 (플레이스홀더)
- **우선순위:** 보통 (향후 확장)

## 3. 비기능 요구사항

### NFR-01: 성능
- 앱 초기 로딩 시간 2초 이내
- 페이지 전환 애니메이션 60fps 유지
- Hive 읽기/쓰기 50ms 이내

### NFR-02: 사용성
- UI 언어: 한국어
- 통화: 원화(₩), 천 단위 콤마 포맷
- Material 3 디자인 가이드라인 준수
- Neo-Minimal Fintech 스타일 일관 유지

### NFR-03: 테스트
- 단위/위젯 테스트 64건 이상 유지
- 테스트 실행 시간 5초 이내
- FakeBox 패턴으로 Hive 의존성 격리

### NFR-04: 호환성
- iOS 12.0+, Android SDK 21+
- Dart SDK >=3.0.6 <4.0.0

## 4. 데이터 모델 요구사항

### Transaction
| 필드 | 타입 | 설명 |
|------|------|------|
| id | String | UUID |
| title | String | 상호명 |
| date | DateTime | 거래 일시 |
| amount | int | 금액 (원) |
| type | TransactionType | expense / income |
| icon | IconData | 카테고리 아이콘 |
| iconColor | Color | 아이콘 색상 |
| category | String? | 카테고리명 (선택) |

### EnglishWord
| 필드 | 타입 | 설명 |
|------|------|------|
| id | String | 고유 식별자 |
| english | String | 영어 단어/표현 |
| korean | String | 한국어 뜻 |
| exampleEn | String | 영어 예문 |
| exampleKo | String | 한국어 예문 |
| category | WordCategory | 9개 카테고리 중 하나 |

### LearningProgress
| 필드 | 타입 | 설명 |
|------|------|------|
| wordId | String | 단어 ID (FK) |
| correctCount | int | 정답 횟수 |
| totalAttempts | int | 총 시도 횟수 |
| lastStudied | DateTime | 마지막 학습일 |
| isBookmarked | bool | 북마크 여부 |

## 5. 제약사항

- 네트워크 미사용 (오프라인 전용)
- 서버/클라우드 동기화 없음
- OCR 엔진 미구현 (image_picker만 연동)
- 단어 데이터 하드코딩 (서버에서 가져오지 않음)
