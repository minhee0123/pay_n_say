**프로젝트명:** Pay N Say
**버전:** 1.0
**작성일:** 2026-03-18

---

## 1. 프로젝트 개요

### 1.1 앱 이름
**Pay N Say** — "지불하고(Pay) 말하다(Say)"
소비 내역을 기록하면서 영어로 표현(Say)하는 것을 의미한다.

### 1.2 비전
일상 소비 활동을 영어 학습의 기회로 전환하여, 가계부 관리와 영어 실력 향상을 동시에 달성한다.

### 1.3 핵심 가치
- **실용성:** 실제 소비 데이터 기반의 맞춤형 영어 학습
- **접근성:** 오프라인 동작, 네트워크 불필요
- **간편성:** 직관적 UI, 최소한의 조작으로 기록/학습

## 2. 대상 사용자

### 2.1 페르소나

#### 페르소나 A: 직장인 김지은 (28세)
- 매달 가계부를 쓰려 하지만 지속하기 어려움
- 영어 공부를 하고 싶지만 시간이 없음
- 출퇴근 중 짧은 시간에 학습하고 싶음

#### 페르소나 B: 대학생 박민수 (22세)
- 용돈 관리가 필요한 대학생
- 취업을 위해 영어 공부 중
- 소비/금융 관련 영어 표현이 부족함

### 2.2 사용 시나리오
1. 커피숍에서 결제 후 → 앱에 "스타벅스 ₩6,500" 기록 → "I spent ₩6,500 on coffee" 학습
2. 월급날 → "3월 급여 ₩2,500,000" 기록 → "I earned ₩2,500,000 from salary" 학습
3. 출퇴근 중 → 단어장에서 금융 용어 복습 → 퀴즈로 테스트

## 3. 핵심 기능

### 3.1 가계부 관리
| 기능 | 설명 | 차별점 |
|------|------|--------|
| 월간 캘린더 | 한눈에 월별 지출 확인 | 날짜별 dot marker로 시각화 |
| 수입/지출 요약 | 월간 잔액 실시간 계산 | 그래디언트 헤더로 한눈에 확인 |
| 직접 입력 | 간편한 수동 기록 | 카테고리 자동 아이콘 매칭 |
| 수정/삭제 | 유연한 내역 관리 | 롱프레스 삭제, 탭 수정 |
| 로컬 저장 | 오프라인 데이터 보존 | Hive NoSQL, 빠른 읽기/쓰기 |

### 3.2 영어 학습
| 기능 | 설명 | 차별점 |
|------|------|--------|
| 가계부 연동 표현 | 실제 지출 → 영어 문장 | 나의 소비가 곧 교재 |
| 소비/금융 단어장 | 38개 실용 단어 | 카테고리별 분류, 예문 포함 |
| 4지선다 퀴즈 | 한↔영 혼합 출제 | 거래 데이터 기반 문제 포함 |
| 학습 진도 추적 | 정답 3회 = 학습 완료 | Hive 영속화, 앱 재시작 후 유지 |
| 북마크 | 중요 단어 표시 | 단어장에서 즉시 토글 |

### 3.3 OCR 영수증 스캔 (향후)
- 카메라/갤러리에서 영수증 이미지 선택
- OCR 엔진 연동 후 자동 내역 입력
- 현재 image_picker 연동까지 완료

## 4. 화면 구성

### 4.1 화면 목록 (8개)

```
① 가계부 메인 (/)
   - 월간 캘린더 + 요약 헤더

② 일별 상세 (/day/:date)
   - 거래 목록 + 요약 카드

③ 직접 입력 (/add)
   - 지출/수입 토글 + 입력 폼

④ 내역 수정 (/edit)
   - 기존 값 채운 입력 폼

⑤ OCR 스캔 (/ocr)
   - 카메라/갤러리 선택

⑥ 영어 학습 대시보드 (/english)
   - 진도 + 오늘의 표현 + 메뉴

⑦ 단어장 (/english/words)
   - 카테고리 필터 + 단어 카드

⑧ 퀴즈 (/english/quiz)
   - 10문제 4지선다 + 결과
```

## 7. 기술적 차별점

1. **오프라인 퍼스트:** 네트워크 없이 모든 기능 동작
2. **가계부-학습 연동:** 실제 소비 데이터가 학습 자료로 자동 변환
3. **경량 아키텍처:** 서버 없이 Hive 로컬 DB만으로 데이터 관리
4. **테스트 안정성:** FakeBox 패턴으로 64개 테스트 2초 내 실행
5. **확장 가능 구조:** Feature-based 모듈로 기능 추가 용이
